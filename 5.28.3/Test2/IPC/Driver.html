NAME

Test2::IPC::Driver - Base class for Test2 IPC drivers.

SYNOPSIS

  1.     package Test2::IPC::Driver::MyDriver;
  3.     use base 'Test2::IPC::Driver';
  5.     ...

METHODS

  • $self->abort($msg)

    If an IPC encounters a fatal error it should use this. This will print the message to STDERR with 'IPC Fatal Error: ' prefixed to it, then it will forcefully exit 255. IPC errors may occur in threads or processes other than the main one, this method provides the best chance of the harness noticing the error.

  • $self->abort_trace($msg)

    This is the same as $ipc->abort($msg) except that it uses Carp::longmess to add a stack trace to the message.

  • $false = $self->use_shm

    The base class always returns false for this method. You may override it if you wish to use the SHM made available in Test2::API/Test2::API::Instance.

LOADING DRIVERS

Test2::IPC::Driver has an import() method. All drivers inherit this import method. This import method registers the driver.

In most cases you just need to load the desired IPC driver to make it work. You should load this driver as early as possible. A warning will be issued if you load it too late for it to be effective.

  1.     use Test2::IPC::Driver::MyDriver;
  2.     ...

WRITING DRIVERS

  1.     package Test2::IPC::Driver::MyDriver;
  2.     use strict;
  3.     use warnings;
  5.     use base 'Test2::IPC::Driver';
  7.     sub is_viable {
  8.         return 0 if $^O eq 'win32'; # Will not work on windows.
  9.         return 1;
  10.     }
  12.     sub add_hub {
  13.         my $self = shift;
  14.         my ($hid) = @_;
  16.         ... # Make it possible to contact the hub
  17.     }
  19.     sub drop_hub {
  20.         my $self = shift;
  21.         my ($hid) = @_;
  23.         ... # Nothing should try to reach the hub anymore.
  24.     }
  26.     sub send {
  27.         my $self = shift;
  28.         my ($hid, $e, $global) = @_;
  30.         ... # Send the event to the proper hub.
  32.         # If you are using the SHM you should notify other procs/threads that
  33.         # there is a pending event.
  34.         Test2::API::test2_ipc_set_pending($uniq_val);
  35.     }
  37.     sub cull {
  38.         my $self = shift;
  39.         my ($hid) = @_;
  41.         my @events = ...; # Here is where you get the events for the hub
  43.         return @events;
  44.     }
  46.     sub waiting {
  47.         my $self = shift;
  49.         ... # Notify all listening procs and threads that the main
  50.         ... # process/thread is waiting for them to finish.
  51.     }
  53.     1;

METHODS SUBCLASSES MUST IMPLEMENT

  • $ipc->is_viable

    This should return true if the driver works in the current environment. This should return false if it does not. This is a CLASS method.

  • $ipc->add_hub($hid)

    This is used to alert the driver that a new hub is expecting events. The driver should keep track of the process and thread ids, the hub should only be dropped by the proc+thread that started it.

    1.     sub add_hub {
    2.         my $self = shift;
    3.         my ($hid) = @_;
    5.         ... # Make it possible to contact the hub
    6.     }
  • $ipc->drop_hub($hid)

    This is used to alert the driver that a hub is no longer accepting events. The driver should keep track of the process and thread ids, the hub should only be dropped by the proc+thread that started it (This is the drivers responsibility to enforce).

    1.     sub drop_hub {
    2.         my $self = shift;
    3.         my ($hid) = @_;
    5.         ... # Nothing should try to reach the hub anymore.
    6.     }
  • $ipc->send($hid, $event);
  • $ipc->send($hid, $event, $global);

    Used to send events from the current process/thread to the specified hub in its process+thread.

    1.     sub send {
    2.         my $self = shift;
    3.         my ($hid, $e) = @_;
    5.         ... # Send the event to the proper hub.
    7.         # If you are using the SHM you should notify other procs/threads that
    8.         # there is a pending event.
    9.         Test2::API::test2_ipc_set_pending($uniq_val);
    10.     }

    If $global is true then the driver should send the event to all hubs in all processes and threads.

  • @events = $ipc->cull($hid)

    Used to collect events that have been sent to the specified hub.

    1.     sub cull {
    2.         my $self = shift;
    3.         my ($hid) = @_;
    5.         my @events = ...; # Here is where you get the events for the hub
    7.         return @events;
    8.     }
  • $ipc->waiting()

    This is called in the parent process when it is complete and waiting for all child processes and threads to complete.

    1.     sub waiting {
    2.         my $self = shift;
    4.         ... # Notify all listening procs and threads that the main
    5.         ... # process/thread is waiting for them to finish.
    6.     }

METHODS SUBCLASSES MAY IMPLEMENT OR OVERRIDE

  • $ipc->driver_abort($msg)

    This is a hook called by Test2::IPC::Driver->abort() . This is your chance to cleanup when an abort happens. You cannot prevent the abort, but you can gracefully except it.

  • $bool = $ipc->use_shm()

    True if you want to make use of the Test2::API/Test2::API::Instance SHM.

  • $bites = $ipc->shm_size()

    Use this to customize the size of the SHM space. There are no guarantees about what the size will be if you do not implement this.
